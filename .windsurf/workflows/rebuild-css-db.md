---
description: Regenerate js/fx-css-db.js from every css/borders-*.css file
---

The standalone-CSS exporter needs a pre-baked text copy of every
`borders-*.css` file so it can work when the page is opened directly
from the filesystem (`file://`), where `sheet.cssRules` is blocked by
the same-origin policy.

Run this workflow whenever you add or edit a `css/borders-*.css` file.

1. Regenerate `js/fx-css-db.js` by concatenating every `css/borders-*.css`
   file into a JS template-literal array.
// turbo
```powershell
$out = "// Auto-generated by /rebuild-css-db workflow.  DO NOT EDIT BY HAND.`n// Contains raw text of every borders-*.css so standalone CSS export works`n// from file:// (where sheet.cssRules is blocked by the same-origin policy).`nwindow.FX_CSS_DB = {`n  texts: [`n"
Get-ChildItem css\borders-*.css | Sort-Object Name | ForEach-Object {
  $text = Get-Content $_.FullName -Raw -Encoding UTF8
  $escaped = $text -replace '\\','\\\\' -replace '`','\`' -replace '\$','`$'
  $out += "    // ==== $($_.Name) ====`n"
  $out += "    ``$escaped``,`n`n"
}
$out += "  ]`n};`n"
Set-Content -Path js\fx-css-db.js -Value $out -Encoding UTF8
Write-Host "Wrote $((Get-Item js\fx-css-db.js).Length) bytes to js\fx-css-db.js"
```

2. (Optional) Verify the file parses by loading `index.html` in a browser
   and opening the modal on any page-1 card — the preview panel should
   show real CSS rules instead of a "rules could not be found" note.
